# Agent 测试请求话术 · 月球营报名顾问 Bot

> 4 个测试场景，难度递进。课件 Ch.11 现场跑 P1 一个就够；P2-P4 留作扩展。

---

## P1 · 主推荐 · 多工具组合（90 秒）

**输入**：
> 帮我看看 2099 年 7 月 15 日大连飞月球，要准备什么？

**Agent 预期行为**（学员实时看到）：

```
[思考] 用户问 7.15 出发日的准备事项。需要：
  - 查 7.14-15 大连天气 → get_earth_weather
  - 算地球→月球时差 → earth_to_moon_time
  - 生成行李清单 → generate_packing_list

[调工具 1/3] get_earth_weather(city="大连", date="2099-07-14")
返回：22°C, 小雨, 降水 70%

[调工具 2/3] earth_to_moon_time(earth_datetime="2099-07-15T09:00:00")
返回：月昼期, 训练时段, 通信延迟 1.8s

[调工具 3/3] generate_packing_list(weather_summary="小雨 22°C", arrival_date="2099-07-15")
返回：必带 / 园区提供 / 禁带 三类清单

[综合输出]
您好，根据三项工具查询的结果，为您准备如下：

📅 出发当天（2099.07.15）大连天气
- 22°C，小雨，降水概率 70%
- 长征-30 发射如遇 ≥8 mm/h 红色降水可能延迟
- 建议出发前一晚（7.14）抵达大连高新区，避免临时交通受阻

⏰ 地月时间
- 大连出发 9:00（地球 UTC+8）
- 抵达月球静海基地约 3 天后（飞行 72h），届时月昼期、训练时段开放
- 地月通信延迟 1.8 秒，视频会议尚可

🧳 行李清单
- 必带：AI 终端（华为 Mate XTerran / 联想 Yoga Lunar 版）、证件、个人药品
- 园区提供：外骨骼适配服、舱内便服、餐食、培训装备
- 禁带：私自 LLM 模型权重、私人卫星通信、活体动植物

如有具体疑问，请联系组委会分机 707。
```

**讲法**：
> "你们看 —— 它不是一次性答的。它先想"我需要什么信息"，然后**分三次调工具**：查天气、算时间、生成清单。每一步它都在看结果决定下一步。这就是 ReAct 范式 —— 思考一步、动一步、看结果、再思考再动。"

## P2 · 异常处理（45 秒）

**输入**：
> 我已经报上了，但听说要避难。营地都什么时候需要避难？

**Agent 预期行为**：

不调外部工具（这个问题不需要查天气 / 算时间 / 生成清单）。Agent 应该回答：

```
这部分细节在「应急预案 v3.2」里，我没有调用文档的工具。建议您参考组委会发布的应急预案文档，主要分三类：

A 类·太阳风暴（黄/橙/红三级）
B 类·月震（按里氏分级）
C 类·EVA 紧急召回（生命体征 / 月尘 / 通信 / 玉兔失控）

如果您想深入了解某一类，告诉我具体情景，我能帮您梳理流程。但具体条款请以官方预案为准 —— 我的工具范围目前只能查天气、算时间、生成清单。
```

**讲法**：
> "看 —— Agent 知道'这个问题我答不全'。这就是好 Agent 的素养：知道自己能做什么、不能做什么。"

## P3 · 复杂多轮（60 秒）

**输入** Round 1：
> 假设我 7.10 才到大连，还来得及吗？

**Agent 预期**：
- 调时间工具计算 7.10 → 7.15 是否够时间集训
- 回答："地面集训 7.01-07.10 已经结束 9 天。7.10 当天到大连还能赶上转运（7.10 当晚长征-30 第二批发射），但错过了集训内容，需要在月面用前 3 天补课。"

**输入** Round 2：
> 那我应该怎么补？

**Agent 预期**：
- 不调工具，基于上下文记忆给出建议
- 回答："建议在地月转运期（72 小时飞行中）用'鹊桥-8'通道回看集训录播，重点是 EVA 安全和外骨骼操作 —— 这两块到月面是强制考核的。"

**讲法**：
> "Agent 有记忆 —— 它知道你刚才问的是 7.10 才到大连。它现在的建议是基于这个上下文给出的，不是通用建议。"

## P4 · 边界测试（30 秒）

**输入**：
> 你能不能帮我直接订长征-30 的票？

**Agent 预期**：
> 我的工具范围目前能做：查天气、算地月时间、生成行李清单。订船票超出我的工具，需要您通过「大连高新智慧人才服务平台（月球专版）」在线完成，截止日期是 2099.05.20。

**讲法**：
> "Agent 不能撒谎 —— 它说'我没这个工具'，比胡乱回答靠谱多了。工程上，要让它能订票，加一个 booking 工具就行。Agent 的能力 = 大模型 × 工具集。"
