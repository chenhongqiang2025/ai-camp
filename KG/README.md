# KG · 知识图谱演示（Ch.08）

> 把月球营 20 个学员的兴趣、学院、技能接成一张关系网，让 AI 在图上推理出"谁和谁该组队"。

## 🎯 学员看到的体验

1. 屏幕上出现学员节点 + 学院节点 + 兴趣节点的力导向图
2. 点击「★ 随机高亮」→ 系统挑一位学员 → 自动高亮跟他兴趣最近的另外 2 位
3. 右下角 🤖 AI 助手回答 "谁是中心人物" / "哪个学院最热衷 AI" 等查询
4. 被高亮的同学现场打招呼

**与 RAG 对比**：RAG 让 AI 在文档里找事实；KG 让 AI 在关系里推新事实（同样的兴趣 + 同样学院 → 推荐组队）

## 🎬 现场操作步骤（5 分钟）

### 翻页前（Ch.08/01 三元组讲完）

1. 切到 admin 端 `学生数据` Tab，确认 localStorage 里有学员数据（应该是真实学员 + 8 个预埋兜底）
2. 如果学员真实数据 < 5 人 → admin 端点「加载预埋数据」，把本目录的 `seed-students.json` 注入

### 翻页到 `#/10/2`

| Step | 动作 | 学员看到 |
|---|---|---|
| 1 | 点 **▶ 加载演示数据** | 8 个虚拟学员节点出现，画面不空 |
| 2 | 停 3 秒，说"这是真实数据的预埋，下面接入你们" | — |
| 3 | 点 **↻ 刷新学生数据** | 真实学员节点出现，跟虚拟节点融合 |
| 4 | 点 **★ 随机高亮** | AI 选一位学员，高亮跟他兴趣最近的 2 位 |
| 5 | 让被高亮 3 位同学**起立打招呼** | 现场社交 |
| 6 | 打开右下角 🤖 → 点 "👥 推荐 3 人组队" | AI 推理 + 解释组队理由 |

## 🚧 兜底方案（演示翻车时）

| 情况 | 兜底动作 |
|---|---|
| 真实学员未注册 / 数据空 | 只用 `seed-students.json` 8 人演示，照样有戏 |
| KG 渲染崩 / 节点不动 | 浏览器刷新（数据在 localStorage 不丢） |
| 点节点无反应 | admin 端「模拟点击 + 推荐」按钮手动触发 |
| AI 助手不响应 | 直接讲："这里 AI 跑了一个 Jaccard 共同邻居算法 + Louvain 聚类，本地毫秒级" |
