# MCP 测试请求话术 · Claude Desktop + filesystem-server

> 4 个请求，从「能不能调工具」到「能不能跨文件总结」，递进展示。
> 前提：`月球营档案/` 文件夹已建好，包含 `营地规则.md` / `训练计划-第一周.md` / `应急预案.md`。

---

## P1 · 「能不能调工具」(20 秒)

**输入**：
> 请用 filesystem 工具，列出我桌面上「月球营档案」文件夹里有哪些文件。

**Claude 预期回答**：
> 根据 filesystem 工具读取，「月球营档案」目录下包含：
> - 营地规则.md
> - 训练计划-第一周.md
> - 应急预案.md

**现场要做的**：指着 Claude 输出中的 `Read directory` 工具调用标识 → "看到了吗，这不是我的输入，是 Claude 真的去调用了 filesystem 工具读了我电脑。"

---

## P2 · 「能不能读内容」(30 秒)

**输入**：
> 用 filesystem 读 `营地规则.md`，告诉我学员每天有多少算力配额？

**Claude 预期回答**：
> 根据 `营地规则.md` 第三章「训练系统访问规范」：每位学员每天的算力配额是 **8 PFLOPS-hour**，使用"蛟龙-3000"类脑芯片集群。

**讲法**：
> "这是真的读了文件 + 找到具体条款。这不是它'记得'什么，是它'查到'了什么 —— 和 RAG 一样，但通过文件系统而不是向量库。"

---

## P3 · 「能不能跨文件汇总」(50 秒)

**输入**：
> 用 filesystem 读那三个 md 文件，汇总一下：如果遇到 EVA 紧急召回，学员需要走哪些步骤？涉及到哪些其他文档里的规定？

**Claude 预期回答**：
> 基于 `应急预案.md` 和 `营地规则.md` 的交叉信息：
>
> **触发**（应急预案 C 类）：生命体征异常 / 月尘风暴 / 鹊桥-8 中断 / 玉兔-10 失控
>
> **流程**：
> 1. 收到基地广播 + 外骨骼震动提醒
> 2. 立即返回最近气闸室（H1/H2/H3）
> 3. 完成"尘洗"流程（营地规则二·5：月尘暴露后必须 14 分钟尘洗）
> 4. 监控室确认 → 解除警报
>
> **相关时间窗**：每次 EVA 不超过 90 分钟（营地规则二·1），紧急召回不计入正常 EVA 时长。

**讲法**：
> "看 —— Claude 自己跨了两份文件、把信息拼起来。这就是 Agent 的雏形：它不只是答你问的，是规划'我该查哪个文件'然后整合。"

---

## P4 · 「让它真动手」(20 秒)（可选，时间紧可跳）

**输入**：
> 在「月球营档案」文件夹下新建一个文件 `我的笔记.md`，内容写一句"今天学了 MCP"。

**Claude 预期回答**：
> 已通过 filesystem 工具写入 `月球营档案/我的笔记.md`，内容为"今天学了 MCP"。

**现场要做的**：切到文件管理器 → 打开「月球营档案」文件夹 → 学员看到 `我的笔记.md` 真的出现了。

**讲法**：
> "Claude 不只是会读，还会写。这就是 AI 长出了手 —— 它能改你电脑的状态。下一章 Agent 我们会看到，它能用同样方式去调更复杂的工具：发邮件、查数据库、做线上支付。"
