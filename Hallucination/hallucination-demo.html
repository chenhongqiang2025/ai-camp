幻觉 · 大模型不是"知道"答案 · 是"预测"下一个最可能的词

CH.06 · 当模型记忆库里没有事实 · 它就用最像的词凑出一个看起来合理的答案 → 编造
示例 prompt： 采样温度 T = 0.80

逐 token 生成 · 鼠标悬停看每一步候选 · 红色 = 编造

怎么读：
• 模型一次只产出一个 token（"字"或"词块"）
• 每一步从概率最高的 N 个候选里采样
• 红色词 = 训练数据里根本没出现过的"事实" · 但模型预测得很流畅
• 关键问题：流畅 ≠ 真实 · 模型不知道自己在编

采样温度 T：
• T = 0 → 永远选最高概率（确定性、保守、但仍可能错）
• T = 1 → 按概率分布随机采样（多样性）
• T = 2 → 趋近均匀分布（创造性、也最容易编）

第 — 步候选概率

点击响应中任意词 → 看那一步候选
三种典型幻觉：

① 事实虚构
  问"哈利波特作者"→ 给 J.K.罗琳 ✓
  问没读过的小说作者 → 编一个名字

② 引用虚构
  模型会编 ISBN · 编 DOI · 编 arXiv 编号
  格式完全正确 · 但不存在

③ API 虚构
  模型会调一个看起来"应该存在"
  的函数 · 但文档里根本没有

解法：RAG / 工具调用 / 人工校验
→ 不让模型靠"记忆"答 · 让它先查再说
幻觉是 LLM 的本质特征 · 不是 bug · 它就是个"预测下一个词"的概率机器 · 流畅 ≠ 真实
上生产 · 必须配 RAG / 检索 / 校验 / 引用回链 / 人审 · 这是后面 RAG 章节的核心
