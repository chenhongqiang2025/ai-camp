CHAPTER · 02

AI 发展简史

从图灵的疑问，到达特茅斯的命名，到两次寒冬，到机器学习的复兴 —— AI 走了 70 年

IN DESIGN 15 min
1950

图灵的疑问

艾伦 · 图灵发表《Computing Machinery and Intelligence》，提出后来被称为"图灵测试"的判断标准 — 如果一台机器能让你相信它是人，那么它就在某种意义上"思考"

"我们能想象的最远的未来，不过是机器中也涌现出某种属于它们自己的智慧。"
强调：这是一切的起点。图灵当时连"计算机"都没几台，却已经在问"机器能思考吗"。 可以问学生："你觉得 ChatGPT 通过图灵测试了吗？" — 引发思考但不展开。
1956

达特茅斯会议 · AI 正式命名

John McCarthy、Marvin Minsky、Claude Shannon 等学者在达特茅斯学院召开为期 8 周的研讨会 — "Artificial Intelligence" 这个词，是在这次会议上被正式提出的

"我们认为，学习的每一个方面或智能的任何其他特性，原则上都可以被精确描述并由机器模拟。"
强调：这是 AI 的"建国日"。 有趣的事实：当时与会者乐观预言"10 年内机器将能做人能做的所有事"——结果，过了快 70 年。 这是后面"AI 寒冬"的伏笔。
CH.02 · 寒冬与复苏

两次 AI 寒冬 · 一次大复兴

AI 不是一条直线上升，而是经历了过两次"被冷落"才走到今天

第一次寒冬 1974-1980

专家系统的失败

  • 过度乐观预言落空
  • 专家系统规则爆炸，难以维护
  • 政府削减研究资助（Lighthill 报告）
  • 核心矛盾：硬规则无法应对真实世界的不确定性
第二次寒冬 1987-1993

第二轮泡沫破裂

  • LISP 机商业失败
  • 日本"第五代计算机"计划失败
  • 专家系统仍未解决根本问题
  • 资本和媒体全面撤退
历史教训： 每次寒冬都源于"承诺超出能力"。今天的 GenAI 也面临同样的拐点 — 但这次，我们有了真实可用的产品，不只是承诺。
这一页要警示学生：技术发展不是直线。 关键洞察：每次寒冬的根因都是"承诺与能力的差距"——这是 AI 行业的固有风险。 这也是为什么 IBM 强调"可落地"的 GenAI——避免重蹈覆辙。

CHAPTER · 03

机器学习与神经网络

从一个"神经元"开始，到能识别万物的深度网络 —— 数学并不可怕

IN DESIGN 20 min
CH.03 · 01 / 04 · 起点

感知机 · 第一个人工神经元

1957 年 Frank Rosenblatt 提出 — 把生物神经元抽象成一行数学

  • 输入
    x₁ · x₂ · x₃ ... — 一组数字，可以是像素、词的索引、传感器读数本质：把世界翻译成"数"
  • 权重
    w₁ · w₂ · w₃ ... — 每个输入的"重要程度"，由训练得来这是网络"记忆"的全部 — 所谓"AI 模型"就是一堆 w
  • 求和
    z = Σ wᵢxᵢ + b — 加权求和，再加上一个偏置项中学数学就能看懂的式子
  • 激活
    y = σ(z) — 非线性函数（Sigmoid、ReLU），决定神经元是否"激活"没有这一步，整个网络等价于一个线性回归
核心理解： 现代最大的 LLM (~万亿参数)，本质就是 万亿 个这样的"小神经元"叠在一起 — 每一个的数学都不超出中学范围
强调："不要被神经网络吓到 — 单个神经元的数学就这么简单。" 可以画一下：3 个输入接到 1 个神经元 — 黑板上 30 秒能画清楚。 关键洞察：复杂性来自"叠加 + 非线性"，不是某个魔术公式。
CH.03 · 02 / 04 · 学习

反向传播 · 神经网络如何"学"

1986 年 Rumelhart · Hinton · Williams 把它推广开 — 这是 90% 现代 AI 的基础算法

→

输入经过网络，得到预测
→
×
计算误差
预测 vs 真实标签的差距
→
←
反向传播
误差沿网络反向传递
→
↓
更新权重
每个 w 朝降低误差的方向小幅调整
类比： 像你考试错题后 — 回头看看哪一步算错了（反向追溯），下次类似题就微调你的"做题习惯"（更新权重）。数百万次迭代，模型就"学会了"。
类比是教学的关键 — "考试错题" 每个学生都体会过。 可以补充：链式法则是数学基础（高数会学到），但这里不展开公式。 强调：90% 现代 AI 训练算法都是这个套路，只是在它上面加 trick。
CH.03 · 03 / 04 · 看图

CNN · 教计算机"看"

1998 年 Yann LeCun 的 LeNet — 用"卷积"模拟人眼的局部感知

传统全连接

每个像素接每个神经元

  • 参数爆炸：100×100 图像就是 10000 个输入
  • 不利用图像的"空间局部性"
  • 训练慢、效果差
卷积神经网络

用小窗口扫描整张图

  • 卷积层：3×3 窗口提取局部特征
  • 池化层：压缩信息，保留要点
  • 参数共享：同一卷积核扫整图
  • 结果：训练快 10×，准确率高 5-10%
影响： 2012 AlexNet 用 CNN 在 ImageNet 大赛上把错误率从 26% 砍到 15.3%，AI 视觉的"ChatGPT 时刻"。今天的人脸识别、自动驾驶视觉、医疗影像 — 都建立在 CNN 上。
强调："局部性"是 CNN 成功的关键 — 因为图像本身就具有这个属性。 类比：人眼看图也不是"一次性"看，而是局部扫描、组合理解。 AlexNet 突破的不是算法本身（CNN 1998 就有），而是 GPU + 大数据让它能跑出来。
CH.03 · 04 / 04 · 序列

RNN · 教计算机"记忆"

如果 CNN 让计算机"看"，RNN 让计算机"读" — 处理一切有顺序的数据

  • 核心
    循环结构 · 每一步的输出会作为下一步的输入之一网络有了"上下文"概念 — 这是处理语言、语音、时序数据的基础
  • 应用
    机器翻译 · 语音识别 · 文本生成 · 2014-2017 NLP 的主流架构Google Translate 早期就是 LSTM (RNN 的改进版) 做的
  • 短板
    梯度消失 + 顺序计算慢 · 难以记住远处信息，无法并行训练这就是为什么 2017 年 Transformer 一出场就替代了它
承上启下： RNN 教会了我们一件事 — "顺序"在语言中至关重要。Transformer 用一种更聪明的方式（Attention）解决了同样的问题。下一章我们就讲它。
RNN 在今天已经被 Transformer 取代，但理解它有助于理解 Transformer 解决的是什么问题。 关键洞察：序列建模的核心是"上下文" — Transformer 把这件事做到了极致。 下一章 Transformer 是真正的高潮。
CHAPTER · 04

NLP 演进史

让机器"理解"语言：从词袋到 Transformer 的范式跃迁

IN DESIGN 20 min
CH.04 · 01 / 04 · 起点

词袋模型 · 最朴素的开始

2000 年代 NLP 主流方法 — 把一句话当成一袋词，统计频率

优点

简单粗暴有效

  • 实现 10 行代码
  • 可解释（每个词的权重清晰）
  • 当年的垃圾邮件分类、新闻分类基础
致命缺陷

丢失了"顺序"和"语义"

  • "我爱你" 和 "你爱我" 在词袋里完全一样
  • "猫" 和 "犬" 在向量上没有任何相似性
  • 同义词、反义词都识别不出来
  • 遇到长文本，向量稀疏到几乎全是 0
启示： 计算机要真正"理解"语言，必须捕捉两件事 — 顺序（语法）和 语义（词与词的关系）。后面的整个 NLP 史，都是在解这两道题。
强调词袋是"砖头"，今天 RAG 中的关键词检索本质还是改进版词袋。 可以提一句 TF-IDF — 词频 × 反文档频率，给"区分度高的词"更高权重。
CH.04 · 02 / 04 · 突破

Word2Vec · 语义就是几何

2013 年 Mikolov · Google — 用一个浅层神经网络，让每个词变成一个 300 维向量

king − man + woman ≈ queen

这不是巧合 — 这是数学。 Word2Vec 把每个词放到一个 300 维空间，相似的词彼此靠近，
语义关系（性别、复数、动词时态）变成了空间中的"方向"

颠覆： 这是 NLP 第一次让计算机理解"语义"。每一个 Embedding 模型（包括今天 RAG 用的）都建立在这个洞察之上 — 词义可以被数字精确表达。
这一页是 NLP 史上的"惊艳时刻"——一定要让学生感受这个 "wow"。 可以现场让学生猜：tokyo - japan + france = ? (paris) 强调：今天所有 RAG 的"语义检索"本质都是 Embedding 的近邻搜索。
CH.04 · 03 / 04 · Attention

注意力机制 · 让模型自己挑重点

2014 Bahdanau 在机器翻译中引入 — 网络学会了"哪些输入对当前输出最重要"

  • 问题
    翻译 "我 喜欢 在 雨天 喝 茶" · RNN 必须把整句话压缩成一个固定向量再翻译长句子就丢信息 — "雨天" 翻到英文时可能已经被淡忘
  • 想法
    让翻译每个词时，"回头看"原句 · 给每个输入词一个"注意力权重"翻 "tea" 的时候，"茶" 的权重高；翻 "rainy" 时，"雨天" 的权重高
  • 效果
    机器翻译质量飞跃 · 长句不再失真，2015-2017 成为标配这是后来 Transformer 的种子 — Attention 太好用了，不如就用 Attention 做一切
视觉化： 想象一个矩阵 — 行是输出词，列是输入词，每个格子的数值表示"翻这个词时，关注那个词多少" — 这就是 Attention
Attention 是 Transformer 的核心思想 — 这一页要讲清楚。 可以画 attention matrix 的示意图。 强调：Attention 不仅用在翻译，今天所有大模型每生成一个词都在用 Attention 回看上下文。

张三 — 是 — 学生

IN DESIGN 20 min ⭐ 互动演示
CH.11 · 01 / 02 · 从对话到行动

Agent · AI 长出了"手"

对话机器人能答问题 — Agent 能完成任务。差别比你想象的大

普通 Chatbot

"我能告诉你怎么做"

  • 你问："帮我订一张明天去北京的机票"
  • 它答："你可以打开携程，搜索...点击..."
  • 本质：把答案讲给你，让你去做
  • 限制：纯文字 — 没有手脚
Agent

"我帮你做了"

  • 你说："帮我订一张明天去北京的机票"
  • 它：调用航班查询 API → 选最便宜的 → 调用支付 → 完成 → 告诉你订单号
  • 本质：分解目标 → 调用工具 → 报告结果
  • 能力：有"手"（工具）+ 有"脑"（规划）
关键能力 = 工具调用 + 多步规划： Ch.10 的 MCP 让 Agent 有了标准化的"手"；Agent 自己的核心能力是"用思考决定下一步做什么"。这就是 ReAct 范式：Reasoning + Acting 循环。
强调 Agent ≠ 更强的 Chatbot — 是完全不同的能力维度。 ReAct 范式简化讲：思考一步、动一步、看结果、再思考、再动一步。 这是 2026 企业级 AI 落地最热的方向。

CHAPTER · 12

A2A · 多智能体协作

Agent 不再独行 —— 两个或更多 Agent 通过协议交谈、协作、达成目标

IN DESIGN 15 min
CH.12 · 01 / 03 · 为什么

一个 Agent 不够用了

真实世界的任务需要"团队"，AI 也一样

单 Agent 困境

"一个人扛所有事"

  • 提示词膨胀 — 一个 prompt 装不下太多角色
  • 上下文混乱 — 客服 + 工程 + 销售思路撞车
  • 无法并行 — 一步步串行慢
  • 难以调试 — 出错不知道哪个环节
多 Agent 协作

"分工 + 协议沟通"

  • 每个 Agent 专精一个角色（客服 / 研发 / 销售）
  • 各自有独立上下文，思路清晰
  • 可以并行工作，效率提升 3-5×
  • 易于监控和调试 — 看哪个 Agent 出错
类比公司： 一个全能员工 vs 一个分工明确的团队。GenAI 时代的"团队"，由多个 Agent 组成。但他们怎么沟通？这就是 A2A 要解决的问题。
强调："Agent 多了就需要协议"，这是工程必然。 可以举一个真实场景：客户问售前问题 — 售前 Agent 接，但回答涉及技术 — 它要 "@" 技术 Agent 协助。
CH.12 · 02 / 03 · A2A 协议

Google 2025 · A2A 协议

Agent-to-Agent Protocol — 让不同 Agent (甚至不同公司的 Agent) 能互相通信、协作

  • 发布
    Google 2025 年 4 月开源 · 与 MCP 互补 — MCP 解决"Agent ↔ 工具"，A2A 解决"Agent ↔ Agent"不绑定 Google 自家技术栈，是开放协议
  • 核心
    Agent Card · Capability Discovery · Task Lifecycle · 三大机制让 Agent 互认互助每个 Agent 公布自己"能做什么"，另一 Agent 可以找它合作
  • 现状
    2026 已被 50+ 框架采用 · Anthropic、Microsoft、IBM、Salesforce 全部支持和 MCP 一样，正在成为事实标准
架构格局： MCP（Agent ↔ Tool）+ A2A（Agent ↔ Agent）= 完整的 Agent 互联网协议栈。这是 2026 真正在发生的事。
A2A 是 2025 年发布的新协议，最新热点。 可以提一句：MCP + A2A 共同构成 "Agent 互联网" — 未来 Agent 之间可以像微服务一样组合。
